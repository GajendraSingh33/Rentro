## Database Architecture

### Core Tables
- **users**: User information (seekers, owners, admins)
- **pg_listings**: Property/PG listing details
- **inquiries**: Messages sent by seekers to owners
- **messages**: In-app messaging between users
- **bookings**: Booking and reservation information
- **payments**: Payment transaction records
- **reviews**: User reviews and ratings
- **favorites**: User's shortlisted listings
- **admin_content**: FAQ, blog posts, static pages

## API Architecture

### Authentication Layer
- JWT-based authentication
- Role-based access control (RBAC)
- OAuth 2.0 for social login

### Service Layer
- Modular services for business logic
- Separation of concerns
- Reusable utilities and helpers

### Repository Layer
- TypeORM for database operations
- Query optimization
- Transaction management

## Caching Strategy

- **Redis Cache**: User sessions, frequently accessed listings, search results
- **Lazy Loading**: Load data on demand
- **CDN**: Static assets, images, videos

## Security Considerations

- HTTPS/TLS encryption
- SQL injection prevention (parameterized queries)
- XSS prevention (input sanitization)
- CSRF protection
- Rate limiting
- JWT token expiry
- Password hashing with bcrypt

## Scalability

- Horizontal scaling with load balancer
- Database indexing for performance
- Caching layer for frequently accessed data
- Asynchronous processing for heavy operations
- Microservices-ready architecture
